My Code Notes


This project was built using a bit of twitter bootstrap and the Backbone Boilerplate project. Github Project. Special thanks to the boCoup team and the project maintainers. In the past few months the project has continued to evolve and grow.

Kiva Loan Data - Per the Cloudspokes instructions, the code uses a JSON API on the Kiva API to supply loan data. This data changes over time as kiva loans and projects develop, so the data visible on this code will change as well.

Requestb.in - The 'Save Load Pledges' feature uses the requestb.in to dump the users selected loans and pledge amounts. To see the state of the bin, us the following link. http://requestb.in/1i9g7vo1?inspect.

Experimental - The code in this submission uses the Layout Manager for Backbone. It is one of many layout/template solutions for backbone. It offers some interesting features for grouping views into modules within the router. The challenge of rending an HTML table with backbone and the layout manager is visible in the way the views handle the table header. I welcome any feedback on how to better handle this.

Whats missing? - Lots of small stuff, If the kiva API is down, the app really should fail with some type of UI message. The use of the twitter bootstrap code, while included and used, it not a full implementation. Getting it to work with the backbone and jquery libraries would be nice. Finally the use of the collection sort after each change to a models, i.e. pledge amount, seems a little heavy handed.
